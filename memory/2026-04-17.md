# 2026-04-17 - Fixed Form Validation Issues

## Context
Dan reported that the "nama" field was showing as required even after inputting data when creating a Brand.

## Root Cause
1. **Frontend Component Issue**: The `Input.vue` component (and several others like `textarea.vue`, `select.vue`, `checkbox.vue`) were missing proper `v-model` support. They did not bind to the `modelValue` or emit updates, so the Inertia `useForm` state was never updated.
2. **Naming Mismatch**: The Brand table uses `nama_brand` as the column name, but both the controller and the frontend were using `nama`.

## Actions Taken
- Fixed `v-model` support in `resources/js/components/ui/input.vue`, `textarea.vue`, `select.vue`, and `checkbox.vue` using Vue 3.5's `defineModel`.
- Updated `app/Http/Controllers/App/BrandController.php` to use `nama_brand` and added a `unique` validation rule. Removed the `deskripsi` field as it doesn't exist in the database.
- Updated `resources/js/pages/app/admin/master-data/brand/Index.vue` to use `nama_brand` and improved `FormField` usage to properly pass error states and display them through the component.

## Outcome
Fixed the mounting of form data from the UI to the backend, and synchronized the field names with the database schema.
